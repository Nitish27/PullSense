# Plan: Persistence Slice 3 - Review Status Read API

**Goal:** Expose persisted `review_runs` data through a small read API so PullSense can report PR review status and recent run history without relying on GitHub comments alone.
**Phase:** 1 continuation / persistence status surface
**Requires:** `.planning/plans/01-06-PLAN.md`
**Provides:** queryable review status for dashboard and future check-run integration
**Context budget:** 3 tasks, ~90 minutes total

### Task 2: Implement Review Status Store Methods And Route

**Files:**
- Modify: `apps/api/src/app.ts`
- Modify: `apps/api/src/db/review-runs.ts`
- Add: `apps/api/src/routes/review-runs.ts`

**Action:**
Implement a read surface for persisted review runs.

Implement:
- typed repository methods for:
  - `listReviewRunsForPullRequest`
  - `getLatestReviewRunForPullRequest`
- `ReviewRunStore` support for those methods, including noop fallbacks
- a new route:
  - `GET /repos/:owner/:repository/pulls/:pullNumber/review-runs`
- response shape that includes:
  - `latest`
  - `runs`
  - identifiers for the requested PR

DO NOT:
- add worker or webhook behavior changes
- introduce pagination yet
- return raw snake_case database rows to clients

**Verify:**
```bash
pnpm --filter @ai-code-review/api test -- --config vitest.config.ts src/db/review-runs.test.ts src/routes/review-runs.test.ts
pnpm --filter @ai-code-review/api typecheck
# Expected: PASS with the status route backed by persisted review run queries
```

**Done when:**
- [ ] review run repository exposes PR-scoped list/latest queries
- [ ] app registers the status route
- [ ] route returns typed latest/history payloads for a PR
- [ ] empty result returns `latest: null` and `runs: []`
