# Plan: Review Dedup And Summary Upsert Slice

**Goal:** Reduce PullSense noise on repeated PR syncs by updating the existing summary comment in place and deduping grouped inline reviews when the inline findings have not materially changed.
**Phase:** 1 continuation / review-noise reduction milestone
**Requires:** `.planning/plans/01-03-PLAN.md`
**Provides:** Stable summary comment updates and fewer duplicate inline review submissions
**Context budget:** 3 tasks, ~90 minutes total

---

- Modify: `packages/github/src/pull-request-files.ts`
- Modify: `packages/github/src/post-pr-comment.ts`

- Modify: `apps/api/src/workers/review-worker.ts`

### Task 2: Inline Review Dedup With Review Signature

**Files:**
- Modify: `packages/github/src/pull-request-files.ts`
- Modify: `packages/github/src/post-pr-review.ts`
- Modify: `packages/github/src/post-pr-review.test.ts`
- Modify: `apps/api/src/workers/review-worker.ts`
- Modify: `apps/api/src/workers/review-worker.test.ts`

**Action:**
Deduplicate grouped inline reviews by detecting when PullSense would submit the same inline findings again.

Implement:
- hidden inline review marker containing a stable signature of the normalized inline comments
- pull request review list + update review summary client support
- GitHub helper that:
  - lists existing reviews
  - finds the latest PullSense-managed inline review
  - skips creating a new review when the signature is unchanged
  - updates the review summary body when the signature matches but the summary text changed
  - creates a fresh review only when the inline findings materially differ

DO NOT:
- assume submitted inline review comments themselves can be edited or deleted as a group
- suppress new reviews when the inline findings have actually changed
- remove the current grouped review behavior for the first matching submission

**Verify:**
```bash
pnpm --filter @ai-code-review/github test
pnpm --filter @ai-code-review/api test -- --config vitest.config.ts src/workers/review-worker.test.ts
# Expected: PASS with inline review dedupe tests green
```

**Done when:**
- [ ] Duplicate inline reviews are skipped on repeated syncs with the same findings
- [ ] Existing review summary bodies can be updated when safe
- [ ] Worker tests cover the dedupe path
