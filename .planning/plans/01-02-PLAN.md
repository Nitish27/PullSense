# Plan: Gemini Review Summary And PR Comment Slice

**Goal:** Turn PullSense from a webhook/file-fetch backend into a visible PR reviewer by generating a structured AI review with Gemini API free tier and posting one markdown summary comment back to the PR.
**Phase:** 1 continuation / first visible review milestone
**Requires:** `.planning/plans/01-01-PLAN.md`
**Provides:** Real AI-generated review summary visible in GitHub PR conversation
**Context budget:** 3 tasks, ~90 minutes total

---

### Task 1: Reviewer Package With Gemini Structured Output

**Files:**
- Modify: `packages/reviewer/package.json`
- Modify: `packages/reviewer/src/index.ts`
- Create: `packages/reviewer/src/review-pull-request.ts`
- Create: `packages/reviewer/src/review-pull-request.test.ts`
- Modify: `packages/shared/src/github.ts`
- Modify: `packages/shared/src/index.ts`

**Action:**
Build the first real reviewer package around Gemini API free tier.

Implement:
- shared types for:
  - normalized PR file input
  - structured review output shape
  - issue severity values
- reviewer package logic that:
  - accepts PR files plus repo metadata
  - builds a compact review prompt from changed files/patches
  - calls Gemini with a JSON-oriented prompt
  - validates the returned result with `zod`
- a deterministic fallback/error surface if Gemini returns malformed output

DO NOT:
- add RAG/indexing yet
- add inline code comments in GitHub yet
- over-design the schema beyond a single PR summary use case

**Verify:**
```bash
pnpm --filter @ai-code-review/reviewer test
# Expected: PASS with structured review generation tests green
```

**Done when:**
- [ ] Reviewer package exposes a real PR review function
- [ ] Structured output is validated before use
- [ ] Shared review result types exist for downstream formatting
