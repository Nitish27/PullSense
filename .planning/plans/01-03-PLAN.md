# Plan: High-Confidence Inline PR Review Slice

**Goal:** Keep PullSense's existing summary comment, then add a second review phase that submits only high-confidence inline findings as a grouped GitHub pull request review.
**Phase:** 1 continuation / inline review milestone
**Requires:** `.planning/plans/01-02-PLAN.md`
**Provides:** GitHub review UI comments anchored to diff lines, grouped under one review submission
**Context budget:** 3 tasks, ~90 minutes total

---

### Task 1: Reviewer Output For High-Confidence Inline Findings

**Files:**
- Modify: `packages/shared/src/github.ts`
- Modify: `packages/reviewer/src/review-pull-request.ts`
- Modify: `packages/reviewer/src/review-pull-request.test.ts`

**Action:**
Extend the reviewer output schema so Gemini can return a second class of findings intended for inline review comments.

Implement:
- shared types for inline review candidates containing:
  - `file`
  - `line`
  - `severity`
  - `title`
  - `body`
  - `confidence`
- reviewer prompt instructions that:
  - keep the existing summary behavior
  - only emit inline findings when the model is confident
  - use diff line numbers from the changed patch context
- schema validation for the richer output

DO NOT:
- remove or replace the current summary comment behavior
- allow non-high-confidence findings into the inline review path
- add RAG or repository indexing

**Verify:**
```bash

# Expected: PASS with inline review schema tests green
```

**Done when:**
- [ ] Reviewer output includes optional inline review candidates
- [ ] Only validated high-confidence inline candidates are available downstream
- [ ] Existing summary review output remains intact

### Task 2: GitHub Grouped Review Submission And Diff Anchoring

**Files:**
- Modify: `packages/github/src/pull-request-files.ts`
- Modify: `packages/github/src/index.ts`
- Create: `packages/github/src/post-pr-review.ts`
- Create: `packages/github/src/post-pr-review.test.ts`
- Modify: `apps/api/src/workers/review-worker.ts`
- Modify: `apps/api/src/workers/review-worker.test.ts`

**Action:**
Add a second phase that submits a grouped pull request review with inline comments anchored to valid diff lines.

Implement:
- GitHub helper for `POST /pulls/{pull_number}/reviews`
- diff-line anchoring utility that maps reviewer `file + line` findings onto GitHub review comment coordinates
- worker flow that:
  - keeps the current summary PR comment
  - filters to high-confidence inline findings
  - submits one grouped `COMMENT` review when valid inline comments exist
- tests that cover:
  - successful grouped review submission
  - filtering out low-confidence or unanchorable findings
  - preserving summary comment posting even when no inline review is submitted

DO NOT:
- switch entirely away from the summary comment
- post one API request per inline comment when a grouped review works
- add review approval/request-changes actions yet

**Verify:**
```bash
pnpm --filter @ai-code-review/github test
pnpm --filter @ai-code-review/api test -- --config vitest.config.ts src/workers/review-worker.test.ts
# Expected: PASS with grouped review and worker tests green
```

**Done when:**
- [ ] PullSense still posts the summary comment
- [ ] PullSense can also submit one grouped inline review for high-confidence findings
- [ ] Invalid anchors are safely skipped instead of breaking the job
