**Provides:** database env support, schema bootstrap, and `review_runs` repository primitives

### Task 2: Implement DB Env, Bootstrap, And Review Run Repository

**Files:**
- Modify: `apps/api/package.json`
- Modify: `apps/api/src/env.ts`
- Create: `apps/api/src/db/postgres.ts`
- Create: `apps/api/src/db/review-runs.ts`
- Modify: `apps/api/src/server.ts`
- Modify: `apps/api/src/worker.ts`

**Action:**
Implement a minimal PostgreSQL foundation using `pg`, then add `review_runs` schema/bootstrap and repository primitives.

Implement:
- `DATABASE_URL` in the validated API env
- a small Postgres pool factory and shutdown helper
- `ensureReviewRunsTable(client)` that creates a `review_runs` table if it does not exist
- typed `ReviewRunRecord` / `ReviewRunStatus` / `ReviewRunConclusion`
- repository helpers for:
  - `createReviewRun`
  - `getReviewRunById`
  - row-to-model mapping
- server and worker startup bootstrapping that ensures the table exists before the service begins handling traffic/work

DO NOT:
- wire webhook enqueue persistence yet
- add a full ORM or migration framework in this slice
- hide SQL inside stringly-typed inline startup code

**Verify:**
```bash
pnpm --filter @ai-code-review/api test -- --config vitest.config.ts src/env.test.ts src/db/review-runs.test.ts
pnpm --filter @ai-code-review/api typecheck
# Expected: PASS with the new DB foundation in place
```

**Done when:**
- [ ] API env exposes `DATABASE_URL`
- [ ] `review_runs` schema bootstrap exists in a reusable module
- [ ] repository helpers return typed models instead of raw DB rows
- [ ] server and worker startup paths initialize the schema
