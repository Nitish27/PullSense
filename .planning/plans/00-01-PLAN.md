# Plan: Phase 0 Foundations Bootstrap

**Goal:** Establish the first runnable monorepo foundation for the AI code review bot using the chosen Option A stack.
**Phase:** 0
**Requires:** None
**Provides:** Root workspace/tooling, local infra, initial app/package scaffolds, and baseline verification commands
**Context budget:** 3 tasks, ~60 minutes total

---

### Task 1: Root Workspace And Tooling

**Files:**
- Create: `.gitignore`
- Create: `.editorconfig`
- Create: `.nvmrc`
- Create: `package.json`
- Create: `pnpm-workspace.yaml`
- Create: `turbo.json`
- Create: `tsconfig.base.json`
- Create: `docker-compose.yml`
- Create: `.env.example`
- Create: `.github/workflows/ci.yml`
- Create: `.planning/STATE.md`

**Action:**
Create the root monorepo foundation for a `pnpm` + `turbo` workspace.

Implement:
- a root `package.json` with shared scripts for `dev`, `build`, `test`, `lint`, `typecheck`, and infra control
- workspace discovery with `apps/*` and `packages/*`
- shared TypeScript defaults in `tsconfig.base.json`
- a local infra file with PostgreSQL and Redis services
- a root `.env.example` documenting the core env vars needed by Phase 0
- a minimal CI workflow that installs dependencies and runs lint, typecheck, test, and build
- editor and git hygiene files for a greenfield TypeScript monorepo

DO NOT:
- add Phase 1 GitHub App behavior yet
- add repo-indexing logic yet
- introduce unneeded infra services beyond PostgreSQL and Redis
- leave placeholder scripts that do not resolve to real workspace commands

**Verify:**
```bash
test -f package.json && test -f pnpm-workspace.yaml && test -f turbo.json
# Expected: exit 0

test -f docker-compose.yml && test -f .env.example && test -f .github/workflows/ci.yml
# Expected: exit 0
```

**Done when:**
- [ ] Root workspace files exist and are internally consistent
- [ ] Infra file includes PostgreSQL and Redis services
- [ ] Root scripts map to real workspace tasks
- [ ] CI workflow exists for lint, typecheck, test, and build

### Task 3: Dependency Install And Baseline Verification

**Files:**
- Modify: `package.json` (if install flow needs script alignment)
- Modify: workspace `package.json` files only if install/build/test feedback requires correction

**Action:**
Install the workspace dependencies, align any script mismatches found during verification, and prove the scaffold works.

Run and fix until green:
- workspace install
- root lint
- root typecheck
- root test
- root build

If install or verification uncovers structural issues, fix only the minimum required to make the planned foundation work.

DO NOT:
- drift into Phase 1 features
- add extra tooling not required to make the foundation verifiable
- claim success without fresh command output

**Verify:**
```bash
pnpm install
# Expected: lockfile created and install succeeds

pnpm lint
# Expected: exit 0

pnpm typecheck
# Expected: exit 0

pnpm test
# Expected: exit 0

pnpm build
# Expected: exit 0
```

**Done when:**
- [ ] Dependencies are installed successfully
- [ ] Lockfile exists
- [ ] Root lint, typecheck, test, and build all pass
- [ ] The project is ready for Phase 1 implementation
