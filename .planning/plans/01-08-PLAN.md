### Task 2: Implement Best-Effort Check Run Sync

**Files:**
- Modify: `apps/api/src/app.ts`
- Modify: `apps/api/src/db/review-runs.ts`
- Modify: `apps/api/src/routes/webhook.ts`
- Modify: `apps/api/src/workers/review-worker.ts`
- Modify: `packages/github/src/index.ts`
- Modify: `packages/github/src/pull-request-files.ts`
- Add: `packages/github/src/check-runs.ts`

**Action:**
Implement best-effort GitHub Check Runs integration on top of persisted review runs.

Implement:
- GitHub helper functions to create and update installation-scoped check runs
- review run store method to persist `check_run_id`
- webhook flow:
  - create queued review run
  - attempt to create queued check run
  - persist `check_run_id` when successful
  - always enqueue the review job even if check run sync fails
- worker flow:
  - load the review run and update check run to `in_progress` when `check_run_id` exists
  - mark check run `completed/success` with summary when review succeeds
  - mark check run `completed/failure` when review fails
  - swallow check run sync errors only after logging them

DO NOT:
- make check run sync a hard dependency for posting review comments
- add new environment variables
- duplicate check run formatting logic across webhook and worker when one helper function can own it

**Verify:**
```bash
pnpm --filter @ai-code-review/github test -- check-runs.test.ts
pnpm --filter @ai-code-review/api test -- --config vitest.config.ts src/db/review-runs.test.ts src/routes/webhook.test.ts src/workers/review-worker.test.ts
pnpm --filter @ai-code-review/api typecheck
pnpm --filter @ai-code-review/github typecheck
# Expected: PASS with best-effort check run sync wired through webhook + worker
```

**Done when:**
- [ ] queued check run creation happens after queued review run creation
- [ ] successful queued creation persists `check_run_id`
- [ ] worker updates check run lifecycle when a persisted id exists
- [ ] review processing still continues if GitHub checks API is unavailable
