### Task 2: Implement Lifecycle Store Methods And Wiring

**Files:**
- Modify: `apps/api/src/app.ts`
- Modify: `apps/api/src/db/review-runs.ts`
- Modify: `apps/api/src/routes/webhook.ts`

- Modify: `apps/api/src/worker.ts`
- Modify: `apps/api/src/workers/review-worker.ts`
- Modify: `packages/shared/src/github.ts`

**Action:**
Implement a reusable review run store API and wire it into webhook + worker execution.

Implement:
- `ReviewRunStore` abstraction with a noop fallback for tests/app construction
- Postgres-backed store methods for:
  - `createQueuedReviewRun`
  - `markReviewRunInProgress`
  - `markReviewRunCompleted`
  - `markReviewRunFailed`
- `reviewRunId` on `PullReviewJob`
- webhook route creates the queued row first, then enqueues the job with the stored id
- worker lifecycle updates:
  - mark `in_progress` before fetching files
  - mark `completed` with summary/comment/review metadata on success
  - mark `failed` with the thrown error message and rethrow

DO NOT:
- add check run calls yet
- swallow worker errors after marking failure
- duplicate SQL status logic between route and worker modules

**Verify:**
```bash
pnpm --filter @ai-code-review/api test -- --config vitest.config.ts src/db/review-runs.test.ts src/routes/webhook.test.ts src/workers/review-worker.test.ts
pnpm --filter @ai-code-review/api typecheck
# Expected: PASS with lifecycle persistence wired through the queue and worker
```

**Done when:**
- [ ] queued review run creation happens before enqueue
- [ ] jobs include `reviewRunId`
- [ ] success path stores completion metadata
- [ ] failure path stores the error message
